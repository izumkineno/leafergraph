# 工程注意事项

这份文档专门记录跨任务可复用的工程踩坑和维护约定。

它不是方案草案，也不是需求列表；只有已经验证过、值得长期提醒维护者的注意点才写在这里。

## 坐标系

### 右键菜单创建节点不要直接使用 `worldPoint`

- `LeaferGraph` 当前节点布局写回使用的是 `page` 坐标，不是 `world` 坐标。
- 因此右键菜单里的“从注册表添加节点”“粘贴 / 复制副本”“插入动画示例链”这类创建路径，应该优先使用 `context.pagePoint`。
- 只有在 `pagePoint` 不可用时，才回退到 `worldPoint` 或 `containerPoint`。

### 为什么

- 画布经过 `fitView()`、缩放或平移后，`worldPoint` 和节点实际布局坐标不再等价。
- 如果右键创建节点时直接把 `worldPoint` 写进 `createNode({ x, y })`，节点会出现明显的落点漂移。

### 当前已修正的位置

- `packages/context-menu-builtins/src/features/canvas_add_node_feature.ts`
- `packages/context-menu-builtins/src/editing.ts`
- `example/mini-graph/src/graph/example_context_menu.ts`

### 排查同类问题时优先检查

- 事件点拿到的是 `pagePoint`、`worldPoint` 还是 `containerPoint`
- 最终写入 `createNode(...)` / `moveNode(...)` / `paste` 的坐标系是否和节点布局坐标一致
- 问题是否只会在缩放、平移、`fitView()` 之后出现

## 文档维护

### 事实文档和提案文档不要混写

- 当前事实优先写进根 README、包 README 和 `docs/` 下的事实型专题。
- 前瞻性 proposal 统一写进 `docs/架构演进与提案总览.md`。
- 不要把“还没定”的方案夹进现状说明里。

### 发现失效链接时先修根入口

- 根 README 是当前 workspace 的唯一总入口。
- 一旦有包 README、模板 README 或专题文档调整路径，先同步根 README 和 AI 索引。
- 不要让根 README 继续保留指向不存在模板、旧脚本或旧 proposal 文件的导航。
