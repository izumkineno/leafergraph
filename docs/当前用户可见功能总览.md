## 2. 当前可见界面

从用户打开 editor 页面后的直接观感看，当前界面已经不是“只有一个画布”的最小 demo，而是一套分区明确的工作区：

- 顶栏当前提供品牌区、文档摘要、命令工具栏和运行控制按钮。
- 主工作区当前采用“左侧节点库、中间画布、右侧检查器”的三栏结构。
- 左侧节点库当前支持搜索、分组浏览、快速创建，并能触发 hover / focus 预览浮层。
- 中间主画布当前负责节点图展示、视口交互、选区反馈、连线操作和运行态投影。
- 右侧检查器当前至少有“属性检查”和“运行态检查”两个标签页。
- 底部运行控制台当前会展示图级运行链、失败聚合和节点运行时详情。
- 状态栏当前会显示文档、authority、pending、运行态和选区摘要。
- 工作区设置当前承接扩展、authority、偏好和快捷键等设置面板。
- 生产构建当前还会额外产出 `authority-python-host-demo.html`，用于单独进入 Python authority host demo 页面。

## 3. 当前可用交互

从用户操作角度看，当前 editor 已经具备一套比较完整的基础编辑闭环：

- 用户现在可以从节点库搜索节点、按分组浏览节点，并直接快速创建节点。
- 用户现在也可以通过画布右键菜单创建节点，而不必只依赖左侧节点库。
- 右键菜单当前已经分成节点、连线、画布三类，并统一走命令总线语义，而不是各自维护一套独立动作。
- 选区当前支持单选、多选、`Ctrl/Cmd/Shift + 点击` 切换、左键框选、右键命中已选节点时保留当前选区，以及点击空白区域取消选中。
- 节点当前支持拖拽移动、选中反馈、允许时显示 resize 句柄、按约束缩放，以及右键“重置节点尺寸”。
- 节点左上角信号球当前已经接入正式折叠交互，点击后会收缩节点壳、隐藏内容区并刷新关联连线锚点。
- 常用编辑动作当前已经进入统一命令链，包括复制、剪切、粘贴、duplicate、删除、全选、适配视图、撤销和重做。
- 常用快捷键当前已经可用，至少包括 `Delete`、`Ctrl/Cmd+C`、`Ctrl/Cmd+X`、`Ctrl/Cmd+V`、`Ctrl/Cmd+D`、`Ctrl/Cmd+A`。
- 节点内部交互当前不只停留在静态展示，文档已经明确存在 `slider` 和 `toggle` 两类交互 Widget 示例。

## 4. 当前可用连线与运行能力

连线和运行链已经不是规划中的概念，而是用户当前可以直接体验的能力：

- 用户现在可以在画布上创建正式连线，而不是只看静态节点摆放。
- 连线右键菜单当前支持删除连线和重新连接。
- “重新连接”当前已经是正式交互入口：进入后会切到 `crosshair` 模式、显示源端口高亮和预览链路，并支持 `Esc` 取消。
- 图级运行控制当前已经具备 `play / step / stop` 三个正式入口。
- 节点级运行控制当前已经具备 `playFromNode(...)`，适合从单个节点起跑做局部调试。
- 当前内建系统节点里已经包含 `system/on-play` 和 `system/timer`，足以组织一条最小运行链。
- 运行结果当前不会只停留在画布内部，而是会同步投影到检查器、运行控制台和状态栏。
- 右侧检查面板当前已经补上“最近一次执行详情”区块，说明执行态不只是在日志里可见。
- 当前 runtime feedback 已经能进入统一 UI 表达，这意味着本地运行和外部运行态回流正在走同一条展示链。

## 5. 当前可用扩展与接线能力

从“怎么把能力装进 editor、怎么把 editor 接到后端”这个用户视角看，当前已经落地的能力主要有下面这些：

- editor 当前内建一个本地 Bundle 加载面板，并按 `Widget Bundles`、`Node Bundles`、`Demo Bundles` 三类分组管理。
- 本地 bundle 当前通过文件选择器导入，而不是依赖源码 alias 或手改配置。
- 工作区当前已经形成 `Bundle 接入 / 主画布` 两类工作分页，并支持开始 / 停止渲染。
- `node` / `widget` bundle 当前可以同时加载多个，并按启用状态累加能力。
- `demo` bundle 当前可以同时加载多个，但同一时刻只会有一个“当前 demo”。
- 导入 `demo` bundle 当前不会自动切换当前画布，必须显式切换，避免用户误以为导入动作一定会整图替换。
- 本地 bundle 当前会写入 IndexedDB，刷新页面后可以恢复。
- authority 推来的 remote bundle 当前不会写入浏览器持久化，只在当前远端会话生命周期里生效。
- authority 模式下，当前 editor 已经能消费 `authority.document` / `authority.documentDiff` 做文档同步，消费 `runtimeFeedback` 做运行态回流，消费 `frontendBundles.sync` 做前端 bundle 目录同步。
- 当前构建产物里已经包含独立的 `authority-python-host-demo.html` 页面，说明 authority host demo 已经是正式可进入入口，而不是只存在于源码里的测试脚本。

## 6. 当前边界与注意事项

如果从用户视角给当前工程一个更准确的定位，可以把它理解成：

**它已经是一套可编辑、可运行、可扩展、可接 authority 的节点图工作区原型，但还不应该被误读成所有环节都已经完全产品化。**

需要特别注意的边界有这些：

- 当前 editor 已经能作为工作区壳层稳定承接很多能力，但它仍然首先是实验工程，而不是完全定型的产品应用。
- 当前有 history，不代表所有交互都已经完整纳入可逆操作。
- 当前有 authority session，不代表所有远端链路、异常治理和使用体验都已经完全产品化。
- 当前有 bundle loader，不代表外部分发、版本兼容和发布渠道已经彻底定型。
- 当前文档里仍然会同时出现“蓝图 / 组件 / 节点”和代码里的 `demo / widget / node`，这里应优先把后者理解成 editor 接入层槽位，而不是长期核心领域模型命名。
- 阅读这份总览时，最安全的判断方式仍然是“只把本文写到的现状当现状，其它未来路线继续回到计划书和架构文档里理解”。

## 7. 当前结论

如果只看用户今天打开页面后能直接感受到的能力，LeaferGraph 当前已经至少具备下面这条闭环：

- 能看到完整工作区
- 能搜索并创建节点
- 能选中、框选、拖拽、缩放、折叠节点
- 能创建、删除、重连连线
- 能复制、剪切、粘贴、撤销、重做
- 能跑图、单步、停止，也能从单节点起跑
- 能看执行详情、失败聚合和运行状态摘要
- 能加载本地 bundle，也能接 authority 文档和前端 bundle 同步

这也是当前最适合对外描述工程状态的一句话：

**LeaferGraph 已经不是只有底层宿主能力的内部实验代码，而是一套用户可以实际打开、操作、运行和扩展的 Leafer-first 节点编辑器原型。**
