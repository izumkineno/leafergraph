# 后端同步命令与事件清单

## 文档信息

- 当前状态：现状优先，持续维护
- 最近校对：2026-03-27
- 适用范围：authority 真源、`@leafergraph/sync` 根层与 session、以及 editor 当前已接线消费面

## 统计边界

这份文档按“业务领域”整理当前后端同步主线，而不是只统计某一层的 notification。

固定对齐 6 处真源：

- 协议真源：
  [`../openrpc/authority.openrpc.json`](../openrpc/authority.openrpc.json)
- editor authority client 面：
  [`../examples/editor/src/session/graph_document_authority_client.ts`](../examples/editor/src/session/graph_document_authority_client.ts)
- editor 当前 document session：
  [`../examples/editor/src/session/graph_document_session.ts`](../examples/editor/src/session/graph_document_session.ts)
- sync 根层合同：
  [`../packages/sync/src/core/types.ts`](../packages/sync/src/core/types.ts)
- sync OpenRPC 映射：
  [`../packages/sync/src/openrpc/openrpc_outlet.ts`](../packages/sync/src/openrpc/openrpc_outlet.ts)
- sync session 恢复链：
  [`../packages/sync/src/session/sync_session.ts`](../packages/sync/src/session/sync_session.ts)

阅读口径固定如下：

- `authority.*` method / notification 是协议真源。
- `@leafergraph/sync` 根层与 session 代表当前协议无关同步面。
- editor 只作为当前已接线消费面出现，不再作为文档命名中心。
- `storage-recovery / authority-event / resync / ack-snapshot` 是 sync 当前真实存在的内部恢复语义，但不是 authority 顶层 method / notification，也不是当前稳定公共载荷字段。

## 当前结论

截至当前仓库状态，后端同步主线已经可以按 5 类能力收口：

- 文档同步：authority 整图、diff、操作提交与整图替换，以及 sync 根层的 `getSnapshot()`、`SyncCommand`、`SyncAck`、`subscribeDocument(...)`
- 运行时同步：authority `controlRuntime` 与 `runtimeFeedback`，以及 sync 根层的 `runtime.control`、`feedback`、`subscribeRuntimeFeedback(...)`
- 连接、错误与恢复：sync 当前公开的连接状态、错误事件、重同步策略与内部恢复语义
- 前端 bundle 同步：`authority.frontendBundlesSync` 与 editor 当前 bundle 订阅链
- 配套接口：`rpc.discover` 和 editor session 的本地 pending / projection / confirmation

其中，authority 的文档与运行时主线已经和 `@leafergraph/sync`、editor 当前消费面形成三层贯通；`frontendBundlesSync` 仍然停留在协议 / editor 专属链路里；`storage-recovery / authority-event / resync / ack-snapshot` 则只应被理解为 sync 当前实现里的内部恢复语义，而不是新的公开协议字段。
