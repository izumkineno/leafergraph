# 注意事项

这份文档合并了旧 `注意事项.md` 的内容，用来放稳定的跨任务提醒。

## 坐标系

- 右键菜单创建节点时，优先使用 `pagePoint`，不要直接用 `worldPoint`。
- 当前运行时写回节点布局时使用的是 page 坐标。
- 如果菜单动作拿错了坐标系，节点在缩放或平移后会出现落点漂移。

## 先检查什么

- 事件携带的是哪种 point
- 目标变更 API 需要哪种坐标系
- 问题是否只会在 `fitView()`、缩放或平移后出现

## 常见误判

- 只改了 widget 文案，不代表节点执行状态一定变了。
- 只刷新了单节点，不代表相邻连线一定自动重算完成。
- 只修正文档内容，不代表索引链接已经同步更新。

## 文档维护

- 事实写进 `docs/` 里的 canonical 文档。
- 没定下来的方案不要混进现状说明。
- 如果链接过期，先修索引，不要把旧路径扩散出去。
